Detailed Description


template<class TOC> class AtlasTOCAggregator< TOC >

Class to aggregate LOD and stub request operations on multiple Atlas TOCs.

This class does some book-keeping that's needed in order to effectively work with two or more independent Atlas TOCs of the same type. The main situation this is needed for is the opacity and shadow maps in blender terrain, although this class could find use elsewhere.

In addition to combining load requests, it also manages refcounts for virtual stubs (ones that are shared by some or none of the TOCs it is managing) and only unloads a stub on a TOC when all of its virtual children are unloaded.

Priorities are also aggregated - the maximum priority of the virtual children of a stub is used as the actual request priority for that stub.
